The Workshop accessible video player

Captions

Introduction

Captions are essentially a text version of the spoken word in the video. Ideally it is best to caption all video, but if this is not possible, a transcript will provide access to the audio content of the video.

Some points to consider when creating captions.
  • Captions should match exactly what is said in the audio.
  • You should include important background sounds; these are usually put inside square brackets like this: [laughter], [music].
  • When a new speaker speaks for the first time, they should be identified if known. This is done by adding their name or title at the beginning of the caption. Once they have been introduced you do not need to do this again.
  • Tags such as >> signify a new/change of speaker; this can also be emphasised using a different text colour for each speaker.
For further reading, this Closed Captioning Standards document should answer any questions you may have.

Caption files

The Workshop accessible video player supports captions in the Timed Text format. This is an XML formatted text file containing the transcript of the video with time code information telling the player when to show the text.

Caption XML files can be created with any text editor or with a code editor such as Notepad++ which will colour the XML tags making it easier to read.

A simple caption file might look like this.

<tt xml:lang="en" xmlns="http://www.w3.org/2006/04/ttaf1" xmlns:tts="http://www.w3.org/2006/04/ttaf1#styling" >
<head>
</head>
<body>
<div xml:lang="en">			
 <p begin="00:00:00.50" end="00:00:04.00"> >> Clive: I’ve been growing giant pumpkins for about 30 years now.</p>
 <p begin="00:00:04.00" end="00:00:06.00"> >> Prof Sue Hartley: And have they been getting bigger every year.</p>
 <p begin="00:00:6.00" end="00:00:7.00">etc. etc</p>
</div>
</body>
</tt>

Caption styling

Introduction

You can alter the look of the caption text by creating a new style. This can be useful for identifying different speakers, using a different colour for each.

By default all captions are styled to white text, Verdana font at size 14px. To change the style of the captions that are displayed you will need to add a different style tag in the caption XML file.

Creating a style

A caption style is created inside the <head> tags of the caption XML file.

A caption style might look like this.

<head>
 <styling>
  <style id="tts:default" tts:color="#FFFFFF" tts:fontFamily="Verdana" tts:fontSize="14px" tts:fontWeight="normal" tts:textAlign="center"  />
 </styling>
</head>

All the styles must sit inside <styling> tags. Each style is defined using a <style> tag.

Each <style> can have the following attributes:

Attribute Function Required Example
id The id must be unique and must begin with 'tts:'. The default style is defined by 'tts:default'. This will set the style for all captions. Yes id="tts:my_new_style"
tts:color A hex colour code, see this list of HTML colours. Bear in mind that some colours may not be suitable for people with vision conditions. No tts:color="#00FFFF"
tts:fontFamily A font family for the caption text, see this list of font families. No tts:fontFamily="Times"
tts:fontSize A size in pixels for the caption text. No tts:fontSize="16px"
tts:fontWeight A weight for the caption text. Accepted values are 'normal' and 'bold'.
NOTE: If you want to make text within the caption bold, italic or underlined, use the HTML <b>, <i>, <u> tag around the text.		 No tts:fontWeight="bold"
tts:textAlign An alignment for the caption text. Accepted values are 'left', 'center', and 'right'. No tts:textAlign="left"

Applying a caption style

To apply the style you have created you need to add the style id to the <p> tag of the caption you want to style. This is done by adding style="tts:my_new_style", where the 'style="id goes here"' matches the 'id' of the new style you have created.

A styled caption file might look like this.

<tt xml:lang="en" xmlns="http://www.w3.org/2006/04/ttaf1" xmlns:tts="http://www.w3.org/2006/04/ttaf1#styling" >
<head>
<styling>
  <style id="tts:my_new_style" tts:color="#00FFFF" tts:fontFamily="Times" tts:fontSize="16px" tts:fontWeight="bold" tts:textAlign="left"  />
 </styling>
</head>
</head>
<body>
<div xml:lang="en">			
 <p begin="00:00:00.50" end="00:00:04.00" style="tts:my_new_style"> >> Clive: I’ve been growing giant pumpkins for about 30 years now.</p>
 <p begin="00:00:04.00" end="00:00:06.00"> >> Prof Sue Hartley: And have they been getting bigger every year.</p>
 <p begin="00:00:04.00" end="00:00:06.00" style="tts:my_new_style"> >> Oh yes, with the seed selection they are getting bigger every year.</p>
 <p begin="00:00:6.00" end="00:00:7.00">etc. etc</p>
</div>
</body>
</tt>

Each time you want the new style to be used you need to add it to the caption as above.

Troubleshooting captions

If your captions file does not display, here are a few things to check.

  • Check the URL to the caption file is correct.
  • Open the XML file in an internet browser. It will tell you if there are any errors.
  • Check for any tags that are not closed or are missing:
    e.g. <p begin="00:00:00.50" end="00:00:04.00"> text here.
    This will not work as the closing </p> is missing.
  • Check for special characters such as the > or & symbols. These should be the HTML version, see this HTML entities list for more info.
